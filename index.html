Layer 1 · 핵심 시스템 ⭐

Pet LifeOS — 삶을 읽는 5개 엔진

누구나 Claude API를 호출할 수 있습니다. 하지만 펫의 삶을 읽는 orchestration system은 아무나 만들 수 없습니다. 차별화는 AI 호출이 아니라 이 처리 계층(Processing Layer)에 있습니다.

1

Timeline Engine

사진 EXIF 날짜, 병원기록 날짜, 영상 metadata를 자동 정렬해 생애 순서를 복원합니다.

2019 입양 · 2020 첫 산책 · 2022 슬개골 수술 · 2023 제주 여행 → 살아있는 타임라인 생성
2

Event Detection Engine

패턴으로 사건을 포착합니다. 사진 급증 → 여행, 병원 방문 증가 → 건강 변화, 위치 반복 → favorite place.

“하루는 봄철 산책 비율이 가장 높았습니다.”
3

Relationship Engine

진짜 차별화. 사진 속 보호자 proximity, 산책 동행 패턴, 붙어 있는 시간을 분석해 관계의 결을 복원합니다.

“하루는 보호자와 밀착형 관계를 형성했습니다.” (attachment 0.84)
4

Memory Gap Engine USP ⭐

빈칸을 질문합니다. 리텐션·품질 향상·락인을 동시에 달성하는 핵심 기능.

AI: “2020년 여름 사진이 급증했어요. 특별한 여행이 있었나요?” → 사용자: “처음 바다 본 날” → 타임라인·스토리 강화
5

Story Composer grounded only

창작 금지. 근거 있는 생성(grounded generation)만 허용합니다. 자료에 없는 것은 지어내지 않습니다.

입력 evidence: 비 오는 날 사진 반복 + 메모 “흙냄새 좋아함” → 생성: “하루는 비 오는 날 흙냄새 맡기를 좋아했다.”
Layer 3 · UX moat ⭐

스토리 타임라인 — 왼쪽 노드를 눌러보세요

일반 타임라인이 아닙니다. 각 시점은 AI가 발견한 ‘이야기’로 채워집니다. 흩어진 자료가 시간 위에 엮여, 한눈에 보이는 ‘삶의 흐름’이 됩니다.

Layer 5 · Memory Workflow

AI + 사람이 함께 기억을 만든다

현재 아무도 안 하는 USP. AI가 빈칸을 발견하고 질문하면, 사용자의 답이 스토리를 강화합니다. 답을 골라보세요.

Layer 2 · 데이터 해자(moat)

Pet Memory Graph — 쌓일수록 강해진다

단순 사진 저장이 아니라, 시간·관계·감정·건강 축의 구조화된 기억 그래프. 자료가 쌓일수록 엮어내는 품질이 오르는 선순환 + 가족이 함께 쓰는 락인 → 후발주자가 복제할 수 없습니다.

Architecture · 현실

사진을 모두 보관하지 않습니다

AI가 삶을 읽는 데 필요한 기억 정보만 추출하고, 원본 보관은 선택형으로 제공합니다. ‘원본 사진 저장 서비스’가 아니라 ‘사진을 읽고 기억 구조만 남기는 서비스’로 설계해 비용을 방어합니다.

무료 업로드
분석용 임시 보관 — EXIF 날짜·위치·이벤트·유사도만 추출, 원본은 7~30일 후 삭제하거나 저해상도 프리뷰만 유지
유료 결제 후
결과물 중심 보관 — Life Timeline · 라이프북 PDF · 썸네일 · 메타데이터 · 대표 사진만 장기 보관
원본 장기보관
별도 유료 옵션(클라우드 앨범) — 월 구독 또는 저장 용량 과금. egress(반복 다운로드) 비용이 저장보다 더 위험
대용량 업로드
전부 저장하지 말고 먼저 읽고 줄인다 — 중복·흔들림·유사사진 자동 제거 → 대표사진만 선별

Ethics

되살리는 게 아니라, 보존한다

슬픔을 결제 압박에 쓰지 않습니다. 자극이 아니라 위로를 설계합니다.

✓ ‘부활’ 포지셔닝 금지

되살린다고 말하지 않습니다. 초기엔 대화형 AI를 도입하지 않습니다.

✓ 기억 보존 중심

집착·의존을 부추기지 않습니다. 상실 불안을 결제 압박에 쓰지 않습니다.

✓ 삭제권 · 강도 조절

언제든 삭제할 수 있고, 노출 강도는 사용자가 제어합니다. 위치 정보는 타임라인 생성에만 사용됩니다.
